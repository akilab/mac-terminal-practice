Mac Terminal Practice

Terminalを、手で覚える。

Macを使ってUNIXコマンドに慣れるための練習ページです。 まずコマンドの役割とよく使うオプションを確認し、必要な回数だけ打鍵してから次へ進みます。 本物のTerminalで実行する前に、このページで入力の型を作ります。

Safety First

壊さない練習場を作る

Terminalは強力です。最初は専用フォルダの中だけで練習します。 特に削除コマンドは、意味を理解してから使います。

Practice Folder

練習用フォルダでだけ実行する

ホームディレクトリ直下に practice を作り、その中で練習します。業務ファイルがある場所では試しません。

Read Before Enter

Enter前にコマンドを読む

rm、mv、cp はファイルを変化させます。何を対象にしているかを読んでから実行します。

Path Sense

現在地を確認する癖をつける

迷ったらまず pwd。どこにいるかを確認してから、次の操作に進みます。

最初に作る練習場Terminalで実行
cd ~
mkdir practice
cd practice
pwd
注意: この教材では sudo、rm -rf /、システム設定を変えるコマンドは扱いません。 まずは「見る」「移動する」「作る」「コピーする」「名前を変える」「安全に消す」だけに集中します。
How To Practice

読む、打つ、確かめる

各コマンドは同じ構成です。役割を読み、よく使うオプションだけ確認し、必要な回数だけ入力します。 入力欄は本物のTerminalではありません。打鍵練習と理解チェック用です。

  1. 説明を読む

    そのコマンドが「何をするためのものか」を先に押さえます。暗記より用途の理解を優先します。

  2. オプションを少しだけ覚える

    最初から全部は不要です。現場でよく使うものだけに絞ります。

  3. 手で打つ

    入力して「確認」を押します。単発の確認コマンドは少なく、操作に慣れたいコマンドは多めに練習します。

  4. 本物のTerminalで試す

    慣れてきたら練習用フォルダの中で実行します。結果を見て、コマンドとファイル操作を結びつけます。

Next Step

次に覚える候補

ここまでの基本コマンドに慣れたら、開発作業でよく使うコマンドへ進みます。 ただし、急がず「現在地」「対象」「実行結果」を読む癖を保ちます。

Text Power

sed / awk

文字列の置換、列ごとの加工、集計に使います。便利ですが表現力が高いので、基本操作に慣れてから進みます。

System

ps / top / kill

実行中のプロセスを確認し、必要に応じて終了します。開発サーバーが止まらない時などに役立ちます。

Archive & Remote

tar / ssh / scp

ファイルをまとめる、リモート環境へ接続する、ファイルを転送するためのコマンドです。

Permissions

chmod / chown

権限や所有者を変更します。間違えると動作に影響するため、ファイル操作に慣れてから扱います。

Batch

xargs

検索結果や一覧を、別のコマンドの引数として渡します。パイプの次に学ぶと理解しやすいです。

Version Control

git

ソースコードの変更を記録する道具です。Terminal操作に慣れてから始めると理解が速くなります。
