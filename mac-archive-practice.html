Mac Archive Practice

まとめて、軽くして、安全に渡す。

Windowsではzipだけで済む場面が多くても、MacやUNIXでは tar、gzip、du、df がよく出てきます。 まず容量を見て、次に中身を確認し、展開先を作ってから開く流れを練習します。

最初から学ぶ 確認セットへ
MacBook-Air % cd ~/practice/archive-lab
MacBook-Air % du -sh source

MacBook-Air % zip -r source.zip source
adding: source/ ...

MacBook-Air % unzip -l source.zip
Archive: source.zip
Length  Name
58  source/docs/readme.txt

Concept

まとめることと、軽くすることを分けて考える

Windowsのzipは「複数ファイルをまとめる」と「圧縮して軽くする」を一度に行います。UNIXでは、まとめる道具と圧縮する道具が分かれていることがあります。

tar

複数ファイルを1つにまとめる

tar はアーカイブを作る道具です。まずは圧縮なしの .tar として、中身を見てから展開する練習をします。

gzip

単一ファイルを圧縮する

gzip は基本的に1ファイルを .gz にします。標準動作では元ファイルが .gz に置き換わります。

zip

まとめる + 圧縮する

zip はWindows利用者にも馴染みやすい形式です。フォルダを入れる時は -r で再帰的にたどります。

練習場所とサンプルを作る
mkdir -p ~/practice/archive-lab/source/docs
mkdir -p ~/practice/archive-lab/source/logs
mkdir -p ~/practice/archive-lab/restore
cd ~/practice/archive-lab
printf 'Mac archive practice\nzip tar gzip\n' > source/docs/readme.txt
printf '2026-07-05 start\n2026-07-05 archive\n' > source/logs/app.log
printf 'name,role\nakira,learner\n' > source/members.csv
pwd
find source -type f
Size

圧縮の前に、容量を見る

何が大きいか分からないまま圧縮すると、期待より軽くならなかったり、不要なものまで送ったりします。まず du と df で見ます。

フォルダ全体の容量を人間向けに見る
cd ~/practice/archive-lab
du -sh source
1階層下ごとの容量を見る
cd ~/practice/archive-lab
du -h -d 1 source
ディスク全体の空き容量を見る
df -h
大きいファイルを探す入口
cd ~/practice/archive-lab
find . -type f -exec du -h {} + | sort -h | tail -n 10
du と df の違い

du はファイルやディレクトリが実際にディスク上で使っている量、df はディスク全体の空き容量を見ます。du と ls -lh の値は少し違うことがありますが、ざっくり容量を見たい時は du -sh で十分です。

df -h は列を見る

Avail は空き容量、Capacity は使用率、Mounted on は場所です。迷ったら / や /System/Volumes/Data など、自分のホームディレクトリが乗っている行を見ます。

sort -h が使えない時

環境によっては sort -h が使えないことがあります。その場合は、まず find . -type f -exec du -h {} + だけで容量の一覧を確認します。

zip / unzip

Windowsとも渡しやすいzipから始める

zip -r の -r は、ディレクトリの中を再帰的に入れる指定です。展開前には unzip -l で中身を見ます。

ディレクトリをzipにまとめる
cd ~/practice/archive-lab
zip -r source.zip source
ls -lh source.zip
zipの中身を展開せずに見る
cd ~/practice/archive-lab
unzip -l source.zip
展開先を作ってから展開する
cd ~/practice/archive-lab
mkdir -p restore/zip
unzip source.zip -d restore/zip
find restore/zip -type f
その場で unzip source.zip しない理由

unzip は指定しないとカレントディレクトリへ展開します。既存ファイルと名前が重なると上書き確認が出ることがあります。練習でも仕事でも、展開先ディレクトリを作ってから -d で指定すると安全です。

同じ名前のzipを作り直す時

同じ名前の source.zip がある状態で zip -r source.zip source を実行すると、既存zipを更新します。練習で結果を比べたい時は、source-v1.zip、source-v2.zip のように名前を変えると分かりやすくなります。

tar

まずは圧縮しないtarを読む

tar は「まとめる」道具です。c はcreate、t はlist、x はextract、f はfileと覚えます。

オプション意味読み方
ccreateアーカイブを作る。
tlist中身の一覧を見る。
xextract中身を展開する。
ffile対象のアーカイブファイル名を指定する。
圧縮なしのtarを作る
cd ~/practice/archive-lab
tar -cf source.tar source
ls -lh source.tar
tarの中身を展開せずに見る
cd ~/practice/archive-lab
tar -tf source.tar
展開先を作ってからtarを展開する
cd ~/practice/archive-lab
mkdir -p restore/tar
tar -xf source.tar -C restore/tar
find restore/tar -type f
tar も上書きすることがある

tar も展開先に同じ名前のファイルがあると上書きすることがあります。初めて展開する時は、空のディレクトリを作って -C でそこへ展開します。

tar.gz

tarにgzip圧縮を足す

.tar.gz は、tarでまとめたものをgzipで圧縮した形式です。z がgzip圧縮を使う合図です。

作る

tar -czf

c で作る、z でgzip、f でファイル名指定です。

見る / 展開

tar -tzf / tar -xzf

t で一覧、x で展開です。展開時は -C で場所を指定します。

tar.gzを作る
cd ~/practice/archive-lab
tar -czf source.tar.gz source
ls -lh source.tar.gz
tar.gzの中身を展開せずに見る
cd ~/practice/archive-lab
tar -tzf source.tar.gz
展開先を作ってからtar.gzを展開する
cd ~/practice/archive-lab
mkdir -p restore/targz
tar -xzf source.tar.gz -C restore/targz
find restore/targz -type f
gzip / gunzip

gzipは単一ファイル用、と覚える

gzip はディレクトリをまとめる道具ではありません。標準動作では memo.txt が memo.txt.gz に置き換わり、gunzip で戻します。

単一ファイルをgzip圧縮する
cd ~/practice/archive-lab
cp source/docs/readme.txt gzip-sample.txt
ls -lh gzip-sample.txt
gzip gzip-sample.txt
ls -lh gzip-sample.txt.gz
gunzipで元の名前に戻す
cd ~/practice/archive-lab
gunzip gzip-sample.txt.gz
ls -lh gzip-sample.txt
cat gzip-sample.txt
元ファイルが置き換わる点に注意

gzip file.txt を実行すると、通常は file.txt が消えて file.txt.gz になります。元ファイルを残したい時は、先にコピーを作ってから試すか、使える環境では gzip -k file.txt のように -k を付けます。

Safety

送る前と展開前に、ひと呼吸置く

アーカイブは便利ですが、どこへ何が出るかを見ないまま展開すると混乱します。中身、展開先、上書き、怪しいパスの4点を確認します。

1. 場所を見る pwd で今いる場所を確認します。
2. 中身を見る unzip -l、tar -tf、tar -tzf で一覧を見ます。
3. 展開先を作る mkdir -p restore/name のように受け皿を作ります。
4. そこへ展開する -d や -C で展開先を指定します。
展開前の安全確認
cd ~/practice/archive-lab
pwd
ls -lh source.zip source.tar source.tar.gz
unzip -l source.zip
tar -tf source.tar
tar -tzf source.tar.gz
絶対パスや怪しいパスに注意

一覧に /Users/... のような絶対パス、../ を含むパス、知らない大量のファイルが見えたら、すぐ展開せずに送信元へ確認します。受け取ったファイルほど、まず一覧を見る習慣が効きます。

展開には、展開後の容量が必要

.zip や .tar.gz のサイズが小さくても、展開後は大きくなることがあります。送る前は du -sh source、受け取った後は展開先の空き容量を df -h で確認します。

Formats

その他の圧縮形式を見分ける

操作練習は tar と gzip に絞ります。ただし実務では .tar.bz2 や .tar.xz も見かけるため、読み方だけ整理します。

形式よく使う確認コマンド意味
.tar.gz / .tgztar -tzf file.tar.gztarでまとめ、gzipで圧縮した形式。まず覚える基本形です。
.tar.bz2tar -tjf file.tar.bz2tarでまとめ、bzip2で圧縮した形式。j がbzip2の合図です。
.tar.xztar -tJf file.tar.xztarでまとめ、xzで圧縮した形式。J がxzの合図です。
.7z7z l file.7z7-Zip形式です。macOS標準では 7z が入っていないことが多く、別途ツールが必要です。
tar系の形式を一覧確認する例
# gzip圧縮
tar -tzf file.tar.gz

# bzip2圧縮
tar -tjf file.tar.bz2

# xz圧縮
tar -tJf file.tar.xz
まずは .tar.gz を確実に読む

bzip2 や xz は環境によって必要なコマンドが入っていない場合があります。教材では一番よく使う .tar.gz を中心に練習し、他の形式は「見かけた時に調べる入口」として覚えます。

MacとWSLで入っている道具が違うことがある

macOSでは通常 zip / unzip が使えますが、WSLの最小構成では入っていないことがあります。環境差でコマンドが見つからない時は、教材の理解が間違っているとは限りません。

Checklist

迷った時の確認セット

何を作ったか、どれくらい大きいか、どこへ展開するかを順番に見ます。困った時ほど、いきなり展開せず確認から始めます。

やりたいことコマンド見るポイント
フォルダの容量を見るdu -sh source送る前に大きさを把握する。
1階層下の容量を見るdu -h -d 1 sourceどのディレクトリが大きいか見る。
空き容量を見るdf -h展開できるだけの空きがあるか見る。
zipの中身を見るunzip -l file.zip展開前にファイル名とパスを見る。
tarの中身を見るtar -tf file.tar圧縮なしtarの一覧を見る。
tar.gzの中身を見るtar -tzf file.tar.gzz はgzip圧縮の合図。
迷った時の確認セット
cd ~/practice/archive-lab
pwd
df -h
du -sh .
find . -type f | head -n 30
find . \( -name '*.zip' -o -name '*.tar' -o -name '*.tar.gz' \) -type f -exec ls -lh {} +
unzip -l source.zip
tar -tf source.tar
tar -tzf source.tar.gz
mkdir -p restore/check-before-extract
覚える順番

zip はWindowsとも渡しやすい形式、tar はまとめる形式、tar.gz はまとめてgzip圧縮した形式、gzip は単一ファイルの圧縮です。展開前は必ず中身を見て、展開先を作ります。
