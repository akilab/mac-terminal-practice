Shell

Terminalとシェルを分けて考える

Terminalは文字を表示して入力を受け取るアプリです。シェルは入力されたコマンドを解釈して実行するプログラムです。

Terminal

画面と入力欄

macOS標準のTerminal.appやiTerm2など。シェルを動かすための窓です。

Shell

コマンドの解釈役

zsh や bash など。変数、alias、PATH、リダイレクト、パイプを解釈します。

macOS

標準はzsh

現在のmacOSでは、通常ログインシェルは zsh です。古い資料では bash 前提の説明もあります。

自分のシェルを確認する 
echo $SHELL
echo $0
ps -p $$
$SHELL と $0 の違い

$SHELL はログインシェル、$0 は今動いているシェルを示します。普段は同じことが多いですが、手動で bash や zsh を起動すると変わります。

Environment Variables

環境変数は、コマンドに渡される設定

環境変数は、シェルやコマンドが参照する名前付きの値です。ホームディレクトリ、言語設定、PATH、ツールの設定などに使われます。

変数よくある意味確認例

USERログイン中のユーザー名echo $USER
SHELLログインシェルecho $SHELL
PATHコマンドを探すディレクトリの一覧echo $PATH
LANG言語や文字コードの設定echo $LANG
EDITOR標準で使うエディタecho $EDITOR

env

MY_NAME=mac

export の役割

シェル変数は、そのシェル内だけの値です。export すると、そこから起動する子プロセスにも渡される環境変数になります。

Shell Config

シェルの設定ファイルを知る

毎回同じ設定を手で打つのは大変です。zshでは主に ~/.zshrc に、aliasやPATHの設定を書きます。

よく触る

~/.zshrc

新しい対話シェルを開いた時に読み込まれます。普段のalias、PATH、プロンプト設定はここに書くことが多いです。

ログイン時

~/.zprofile

ログインシェルで読み込まれます。Homebrewの初期設定などがここに入ることがあります。

設定ファイルの存在を確認する 
ls -la ~ | grep zsh
ls -la ~/.zshrc ~/.zprofile 2>/dev/null
編集前にバックアップする 
cp ~/.zshrc ~/.zshrc.backup
ls -l ~/.zshrc.backup
設定を反映する 
source ~/.zshrc
exec zsh
設定ファイルは小さく育てる

一度に大量の設定を入れると、どれが原因で壊れたか分かりにくくなります。1つ追加したらTerminalを開き直して確認する、くらいが安全です。

PATH

PATHは、コマンドを探す順番

ls や python3 と入力した時、シェルはPATHに並んだディレクトリを左から順番に探します。

1. 入力python3 と打つ
2. PATHを見る: 区切りの場所を左から確認
3. 最初に見つける見つかった実行ファイルを使う
4. 見つからないcommand not found になる
PATHを見やすく表示する 
echo $PATH
echo $PATH | tr ':' '\n'
コマンドがどこにあるか調べる 
command -v ls
command -v python3
which python3
type python3
自分用のbinをPATHに追加する例 
mkdir -p ~/.local/bin
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
echo $PATH | tr ':' '\n'
順番が大切

$HOME/.local/bin:$PATH は自分のコマンドを先に探します。$PATH:$HOME/.local/bin は最後に探します。同じ名前のコマンドがある時、先に見つかった方が使われます。

HomebrewとPATH

Apple Silicon Macでは /opt/homebrew/bin、Intel Macでは /usr/local/bin がHomebrewの代表的な置き場所です。brew shellenv の案内に従って設定されます。

Alias / Functions

よく使う操作に名前をつける

aliasは短い別名です。長いコマンドや、よく使うオプションつきコマンドを覚えやすくできます。

一時的なaliasを試す 
alias ll="ls -la"
alias grep="grep --color=auto"
alias
ll
aliasを設定ファイルに保存する 
echo 'alias ll="ls -la"' >> ~/.zshrc
echo 'alias c="clear"' >> ~/.zshrc
source ~/.zshrc
少し複雑なら関数にする 
mkcd() {
  mkdir -p "$1"
  cd "$1"
}
aliasは便利だが、教材では元のコマンドも覚える

aliasは自分の環境では便利ですが、他人のMacやサーバーでは存在しないことがあります。まず元のコマンドを理解し、その後にaliasで短くするのがよい順番です。

UNIX Directories

標準的なディレクトリ構成を読む

UNIX系OSには、役割ごとにディレクトリを分ける慣習があります。macOSはUNIXの流れを持ちつつ、独自のディレクトリもあります。

場所役割見る時の例
/bin基本的な一般ユーザー向けコマンドls /bin
/sbinシステム管理向けコマンドls /sbin
/usr/bin多くの標準コマンドls /usr/bin | head
/usr/sbin追加の管理向けコマンドls /usr/sbin | head
/usr/local/binユーザーが追加したコマンドの代表的な場所ls /usr/local/bin
/opt/homebrew/binApple Silicon MacのHomebrewコマンド置き場ls /opt/homebrew/bin
/etcシステム全体の設定ファイルls /etc | head
/varログ、キャッシュ、状態が変わるデータls /var
/tmp一時ファイルls /tmp
/ApplicationsmacOSアプリケーションls /Applications | head
~/LibraryユーザーごとのmacOSアプリ設定やキャッシュls ~/Library
主要ディレクトリを実際に見る 
ls /
ls /bin | head
ls /usr/bin | head
ls /etc | head
ls /var
触ってよい場所、見るだけの場所

練習で作業する場所は基本的に ~/practice やホームディレクトリ配下です。/bin、/sbin、/etc などは、まず「見るだけ」にします。

Troubleshooting

設定で困った時の見方

PATHやシェル設定で困った時は、感覚で直すより、今の状態を小さく確認します。

command not found

コマンドが見つからない

インストールされていない、または実行ファイルの場所がPATHに入っていない可能性があります。

設定が効かない

読み込むファイルが違う

~/.zshrc に書いたか、source ~/.zshrc したか、今のシェルがzshかを確認します。

command not foundの調査 
echo $PATH | tr ':' '\n'
command -v brew
command -v python3
ls -l /opt/homebrew/bin/python3 2>/dev/null
ls -l /usr/local/bin/python3 2>/dev/null
設定ファイルの調査 
echo $SHELL
echo $0
grep -n "PATH" ~/.zshrc
grep -n "alias" ~/.zshrc
source ~/.zshrc
PATHの重複を見つける 
echo $PATH | tr ':' '\n' | sort | uniq -c | sort -nr
まず戻せる状態を作る

設定ファイルを編集する前にバックアップを作る、追加した行にはコメントを書く、うまくいかない時は直前の変更だけ戻す。この3つで多くの事故を避けられます。
