Mac Permission Practice

Permission deniedで、立ち止まらない。

Windows中心で使ってきた人向けに、UNIX/macOSのファイル権限を読み解く教材です。 ls -l、rwx、chmod、実行権限、sudo、所有者とグループを、安全な練習場所で確認します。

権限から学ぶ 確認順を見る
MacBook-Air % ls -l
-rw-r--r-- 1 akilab staff 42 memo.txt
-rw-r--r-- 1 akilab staff 33 script.sh

MacBook-Air % ./script.sh
zsh: permission denied: ./script.sh

MacBook-Air % chmod u+x script.sh
MacBook-Air % ls -l script.sh
-rwxr--r-- 1 akilab staff 33 script.sh
権限とは ls -l rwx chmod 実行権限 sudo 所有者 確認順
Concept

権限とは、誰が何をできるか

UNIXでは、ファイルやディレクトリごとに「読む」「書く」「実行する、または中へ入る」を誰に許すかが決まっています。

User

所有者

そのファイルを持っているユーザーです。多くの場合、自分が作ったファイルの所有者は自分です。

Group

グループ

複数ユーザーをまとめる単位です。macOSでは普段あまり意識しませんが、staff などがよく出ます。

Others

その他

所有者でも、そのグループでもない同じMac上のユーザーや処理です。インターネットへ公開される、という意味ではありません。

安全な練習場所を作る
mkdir -p ~/practice/permission-lab
cd ~/practice/permission-lab
pwd
touch memo.txt
mkdir notes
ls -l
練習はホームディレクトリ配下で

この教材のコマンドは ~/practice/permission-lab の中で試す前提です。/bin、/etc、/usr などのシステム領域では練習しません。

ls -l

長い表示を、左から読む

ls -l は、種類、権限、所有者、グループ、サイズ、更新日時、名前を表示します。まず左端の10文字に慣れます。

ls -lの出力例を作る
cd ~/practice/permission-lab
echo "hello permission" > memo.txt
mkdir -p notes
ls -l
出力例と見る場所
-rw-r--r--  1 akilab  staff  17 Jul  5 10:00 memo.txt
drwxr-xr-x  2 akilab  staff  64 Jul  5 10:00 notes
これは実行するコマンドではありません

上の2行は ls -l の表示例です。Terminalへ貼り付けるものではなく、左端の10文字と所有者・グループ・名前を見比べるために使います。

部分意味読み方
- / d種類- は普通のファイル、d はディレクトリです。
rw-r--r--権限3文字ずつ、所有者・グループ・その他の権限です。
akilab所有者そのファイルを持っているユーザーです。実際には自分のユーザー名が出ます。
staffグループそのファイルに割り当てられたグループです。
memo.txt名前対象のファイル名やディレクトリ名です。
10文字を分ける

-rw-r--r-- は、先頭1文字が種類、残り9文字が権限です。rw-、r--、r-- のように3文字ずつ区切ると読めます。

rwx

r、w、xは対象で意味が変わる

r は読む、w は書く、x は実行です。ただしディレクトリの x は「中へ入る、名前をたどる」意味になります。

権限ファイルの場合ディレクトリの場合
r中身を読める。cat memo.txt など。中の名前一覧を読める。ls notes など。
w中身を書き換えられる。そのディレクトリ内で、ファイルやディレクトリを作成・削除・名前変更する権限です。多くの場合、実際に操作するには x も必要です。
xプログラムやスクリプトとして実行できる。そのディレクトリへ入れる、配下のパスをたどれる。
ファイルとディレクトリを見比べる
cd ~/practice/permission-lab
ls -ld .
ls -l memo.txt
ls -ld notes
ディレクトリのxはとても大切

ディレクトリに x がないと、その中のファイル名を知っていてもパスをたどれません。ファイルを消せるかどうかは、ファイル自身より親ディレクトリの権限が大きく関係します。通常は親ディレクトリに w と x があると、その中でファイルやディレクトリを作成・削除・名前変更できます。

ls -ld . の読み方

. は今いるディレクトリです。ls -ld . は、中身の一覧ではなく「このディレクトリ自身の権限」を表示します。

chmod

chmodは、まず記号形式で覚える

chmod は権限を変更するコマンドです。初心者は、いきなり数字で丸ごと置き換えるより、u+x のような記号形式から始めると安全です。

誰に

u g o a

u は所有者、g はグループ、o はその他、a は全員です。

どうする

+ - =

+ は追加、- は削除、= は指定した権限に置き換えです。

何を

r w x

読む、書く、実行する権限です。複数まとめて rw のように書けます。

記号形式で変更する
cd ~/practice/permission-lab
touch practice.txt
ls -l practice.txt
chmod u+x practice.txt
ls -l practice.txt
chmod go-r practice.txt
ls -l practice.txt
chmod u=rw,go=r practice.txt
ls -l practice.txt
実行権限は主にスクリプトや実行ファイル向け

ここでは表示の変化を見るために practice.txt へ x を付けています。普段、実行権限を付けるのは script.sh のようなスクリプトや、実行ファイルが中心です。

指定意味よくある使い道
chmod u+x script.sh所有者に実行権限を追加自分のスクリプトを実行できるようにする。
chmod go-r file.txtグループとその他から読み取り権限を外す他の人に読ませたくないファイルに近づける。
chmod u=rw,go=r file.txt所有者は読み書き、他は読み取りだけにする普通のテキストファイルに近い状態へ戻す。
数字形式表示代表的な意味
755rwxr-xr-x所有者は実行まで可能、他は読み取りと実行。公開するディレクトリや実行ファイルで見かけます。
644rw-r--r--所有者は読み書き、他は読み取り。普通のファイルでよく見ます。
600rw-------所有者だけ読み書き。秘密鍵や個人用設定でよく見ます。
数字は r=4、w=2、x=1 の足し算

7 は 4+2+1 なので rwx、6 は 4+2 なので rw-、5 は 4+1 なので r-x です。755 は「所有者7、グループ5、その他5」と読みます。

chmod 777 は安易に使わない

777 は全員に読み取り・書き込み・実行を許す設定です。エラーが消えることはありますが、誰でも変更できる状態に近づきます。原因を読まずに chmod 777 で解決する習慣は避けましょう。

Executable

スクリプトは、読むだけでは実行できない

./script.sh として直接実行するには、ファイルに実行権限が必要です。Permission denied は「存在しない」ではなく「実行してよい権限がない」場合にも出ます。

Permission deniedを安全に再現する
cd ~/practice/permission-lab
printf '#!/bin/zsh\necho "hello from script"\n' > script.sh
ls -l script.sh
./script.sh
所有者に実行権限を追加する
chmod u+x script.sh
ls -l script.sh
./script.sh
実行権限なしでもシェルに読ませれば動く例
chmod u-x script.sh
zsh script.sh
ls -l script.sh
./script.sh と zsh script.sh の違い

./script.sh はファイル自体を実行します。zsh script.sh はzshにファイルを読ませます。前者には実行権限が必要で、後者は読み取り権限が中心です。

sudo

sudoは、権限の意味を理解して使う

sudo は管理者権限でコマンドを実行する仕組みです。便利ですが、何でも sudo で突破すると原因が見えなくなります。

使う前

対象を確認する

pwd、ls -l、whoami で、どこで誰として何を触ろうとしているか見ます。

使った後

所有者が変わることがある

sudo で作ったファイルは root 所有になることがあります。あとで自分が編集できない原因になります。

sudoの前に確認する情報
pwd
whoami
id
ls -ld .
ls -l
「Permission denied なら sudo」は危険な近道

sudo が必要な場面はありますが、作業場所が間違っている、所有者が違う、実行権限がない、ファイルではなくディレクトリ権限が足りない、という原因もあります。まず状態を読みます。

Owner / Group

所有者とグループは、読み方を先に覚える

id、whoami、ls -l で、自分が誰で、ファイルが誰のものかを確認します。所有者やグループの変更は管理作業に近いので、まず読み方を覚えます。

自分のユーザーとグループを見る
whoami
id
groups
ls -l ~/practice/permission-lab
コマンド役割初心者向けの扱い
whoami今のユーザー名を表示する安全に確認できます。
idユーザーIDと所属グループを表示する安全に確認できます。
chown user file所有者を変更する管理作業向けです。教材では読み方中心にします。
chown user:group file所有者とグループを同時に変更するchown だけでグループも指定できます。
chown :group fileグループだけを変更する所有者を変えず、グループだけ変える書き方です。
chgrp group fileグループを変更するchown :group file と似た用途です。環境や手順書に合わせて使います。
chown/chgrpの形だけ読む
# 読み方の例です。むやみに実行しません。
# chown akilab memo.txt
# chown akilab:staff memo.txt
# chown :staff memo.txt
# chgrp staff memo.txt
# ls -l memo.txt
chown -R は特に慎重に

-R は配下すべてに再帰的に変更をかけます。場所を間違えると広い範囲の所有者を壊します。この教材では実行例として推奨しません。必要な時は対象ディレクトリを pwd と ls -ld で確認してから、経験者や手順書に従います。

Troubleshooting

権限で困った時の確認順

焦って権限を広げる前に、対象、ユーザー、権限、所有者、操作内容を順番に確認します。

1. 場所pwd で作業場所を確認
2. 対象ls -l / ls -ld で見る
3. 自分whoami / id で確認
4. 最小変更chmod u+x など必要分だけ
状況まず見るもの次に考えること
./script.sh が Permission deniedls -l script.sh所有者に x がなければ chmod u+x script.sh。
ファイルを編集できないls -l file と whoami所有者が自分か、w があるかを見る。
ディレクトリへ入れないls -ld directoryディレクトリに x があるかを見る。
sudo が必要そうに見えるpwd と対象パスシステム領域を触っていないか、作業場所が正しいか確認する。
よく分からないので 777 にしたい変更したい理由全員に書き込みを許す前に、必要な相手と必要な権限だけに絞る。
権限で困った時の確認セット
pwd
whoami
id
ls -ld .
ls -l
# 対象がディレクトリなら
ls -ld 対象ディレクトリ名
# 対象がファイルなら
ls -l 対象ファイル名
解決の基本は、必要な分だけ変えること

chmod 777、sudo、chown -R は、雑に使うと問題を隠したり広げたりします。まず読み、原因を絞り、chmod u+x script.sh のように最小限だけ変えるのが安全です。

Mac Permission Practice. 権限を読んで、必要な分だけ安全に変更できるようにします。
