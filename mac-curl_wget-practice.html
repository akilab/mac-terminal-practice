Setup

練習場所とwgetを準備する

Macには通常 curl が入っています。一方で wget は入っていないことが多いので、 Homebrewで追加してから練習します。

練習フォルダを作るTerminalで実行
mkdir -p ~/practice/curl-wget-practice/downloads
cd ~/practice/curl-wget-practice
wgetがない場合Homebrew
brew install wget
wget --version
curl --version
この教材の前提: 作業場所は ~/practice/curl-wget-practice です。 取得先には、練習用のApacheログとHTTP確認用の公開テストサービスを使います。 認証情報や社内URLは入力しません。
wget first

先にwgetから覚える理由

wget は「URLからファイルを保存する」目的がはっきりしています。 curl は保存もできますが、HTTPリクエストを細かく作る道具でもあるため、後半で扱います。

wget

ファイル保存が得意

wget URL だけで、URL末尾の名前でファイル保存します。再開や保存先指定も分かりやすいです。

curl

HTTP操作が得意

画面表示、保存、ヘッダー確認、POST、認証ヘッダーなど、HTTPリクエストを細かく作れます。

Mac

curlは標準、wgetは追加

Macでは curl が最初から使えることが多く、wget はHomebrewで入れるのが一般的です。

Options

wgetとcurlの基本オプション

まずはよく使うものだけ覚えます。wget は保存、curl はHTTPの確認と送信に寄せて整理します。

コマンドオプション意味例
wget-O保存ファイル名を指定するwget -O apache_logs URL
wget-P保存先フォルダを指定するwget -P downloads URL
wget-c途中までのダウンロードを再開するwget -c URL
wget--spider保存せず存在確認するwget --spider URL
curl-o保存ファイル名を指定するcurl -L -o apache_logs URL
curl-OURL末尾の名前で保存するcurl -L -O URL
curl-Lリダイレクトをたどるcurl -L URL
curl-Iヘッダーだけ見るcurl -I URL
curl-s進捗表示を抑えるcurl -s URL
curl-w結果の情報を指定形式で表示するcurl -w "%{http_code}\n" URL
curl-Hヘッダーを追加するcurl -H "Accept: application/json" URL
curl-X / -dメソッドと送信データを指定するcurl -X POST -d "name=mac" URL
URLは引用符で囲む: ? や & を含むURLは、シェルが別の意味として解釈することがあります。 練習では "https://example.com/path?name=mac&team=dev" のように囲みます。
Final Drill

取る、見る、確かめる

wget はファイル取得の入口、curl はHTTP確認の入口です。 使い分けられると、ログ教材の準備、API調査、疎通確認が一気に楽になります。

仕上げの例wget / curl
wget -O apache_logs "https://raw.githubusercontent.com/elastic/examples/master/Common%20Data%20Formats/apache_logs/apache_logs"
curl -L -o apache_logs "https://raw.githubusercontent.com/elastic/examples/master/Common%20Data%20Formats/apache_logs/apache_logs"
curl -I https://example.com
curl -s -o /dev/null -w "%{http_code}\n" https://example.com
