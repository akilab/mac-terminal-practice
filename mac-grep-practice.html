Setup

検索用の材料を準備する

Apacheログに加えて、複数ファイル検索用の小さなログ群を作ります。 前回の apache_logs がある人は、そのまま使えます。

Apacheログを用意する未取得の場合
mkdir -p ~/practice/grep-practice
cd ~/practice/grep-practice
curl -L -o apache_logs "https://raw.githubusercontent.com/elastic/examples/master/Common%20Data%20Formats/apache_logs/apache_logs"
複数ファイル検索用の小さなログを作るTerminalで実行
mkdir -p grep-lab
cat > grep-lab/app.log <<'LOG'
INFO application started
WARN disk usage high
ERROR database timeout
INFO application stopped
LOG
cat > grep-lab/api.log <<'LOG'
GET /api/users 200
POST /api/login 401
GET /api/items 200
error invalid token
LOG
cat > grep-lab/security.log <<'LOG'
ALLOW 203.0.113.10 /api/users
DENY 198.51.100.23 /admin

ALLOW 203.0.113.10 /api/items

この教材の前提: 作業場所は ~/practice/grep-practice、Apacheログのファイル名は apache_logs、 複数ファイル検索用ディレクトリは grep-lab/ です。
Options

grepの基本オプション

最初から全部覚える必要はありません。まずは、検索結果を読みやすくし、範囲を広げ、件数を数えるオプションを押さえます。

オプション意味例
-n行番号を表示するgrep -n ERROR grep-lab/app.log
-i大文字小文字を区別しないgrep -i error grep-lab/*.log
-v一致しない行を表示するgrep -v INFO grep-lab/app.log
-c一致した行数を数えるgrep -c DENY grep-lab/security.log
-l一致したファイル名だけ表示するgrep -l error grep-lab/*.log
-rディレクトリ配下を再帰検索するgrep -r DENY grep-lab
-E拡張正規表現を使い、| や + や () を読みやすく書くgrep -E "ERROR|WARN" grep-lab/app.log
-o一致した部分だけ表示するgrep -o -E " 40[0-9] " apache_logs
Regex For Grep

grepで使う小さな正規表現

ここは正規表現そのものを深く学ぶ場ではありません。 grepでログを探す時によく使う、少数の形だけを扱います。

Range

[0-9]

数字1文字に一致します。例: 40[0-9] は400から409のような文字列を探します。

Beginning / End

^ と $

^ERROR はERRORで始まる行、log$ はlogで終わる行に一致します。

Choice

-E と |

grep -E "ERROR|WARN" のように、どちらかに一致する行を探せます。

引用符を使う: 正規表現には |、*、. などシェルにも意味がある文字が出ます。 grepのパターンは、基本的に "..." または '...' で囲みます。
-E がなくても使えるもの: 通常の grep でも、[0-9]、^、$、.、* などは使えます。 -E は正規表現を初めて有効にするオプションではなく、|、+、?、() などをバックスラッシュなしで読みやすく書くためのオプションです。
Final Drill

grepを調査の入口にする

grepは、ログ分析やコード調査の最初の一手です。 見つける、数える、除外する、ファイルを特定する、そしてパイプにつなぐ流れを体に入れます。

仕上げの例grepから分析へ
grep " 404 " apache_logs | wc -l
grep " 404 " apache_logs | cut -d ' ' -f 7 | sort | uniq -c | sort -nr | head
grep -r -i error grep-lab
grep -l -i error grep-lab/*.log
