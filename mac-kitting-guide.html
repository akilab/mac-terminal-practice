Before Start

最初に決めること

Apple AccountやiCloud同期は、使い方に合わせて選びます。 迷う項目は「あとで設定」に寄せ、まずは開発できる状態を作ります。

Apple Account

iCloud同期は目的を決めて選ぶ

iCloud Drive、写真、キーチェーン、Safari履歴などは便利ですが、開発用Macに何を同期するかを先に決めておくと管理しやすくなります。

Local User

ローカルユーザー名を決める

例: 氏名ローマ字、ハンドル名、学習用の短い名前。あとでパスが変わると開発環境の説明がずれるため、最初に決めます。

Security

Touch IDと画面ロック

Touch IDを登録し、スリープ解除時のパスワード要求を有効にします。カフェ作業よりも会議室作業で油断しがちです。

おすすめ方針: Apple Accountは「App StoreやiCloudを使う時にサインイン」。写真、メッセージ、キーチェーン同期は、開発用Macに持ち込みたい情報だけを選びます。
Account Handover

作業用アカウントから移行する

検収や初期作業のために用意された作業用アカウントで最初にログインし、 各自のローカルユーザーを作成してから、作業用アカウントを使わない状態にします。

ここで扱うのはMacのローカルユーザー

Apple AccountはiCloudやApp Storeに使うアカウントです。 ここで作るローカルユーザーは、このMacへログインするためのアカウントです。 まずローカルユーザーを整え、その後でApple Accountの同期有無を判断します。

  1. 作業用アカウントでログインする

    検収用に用意されたユーザーでMacへ入ります。初回ログイン後、システム設定を開けることを確認します。

  2. 自分用のローカルユーザーを作成する

    Appleメニュー → システム設定 → ユーザとグループ → ユーザーを追加。開発端末として自分でツールを入れる場合は、アカウント種別を「管理者」にします。

  3. 新しいユーザーでログインし直す

    作業用アカウントからログアウトし、自分用アカウントでログインします。Touch ID、キーボード、Finder、Dockなどの設定はこのユーザー側で整えます。

  4. 管理者権限を確認する

    システム設定で鍵付きの変更や、ターミナルで管理者パスワードが必要な操作を実行できるか確認します。ここを確認する前に作業用アカウントを消さないでください。

  5. 作業用アカウントを無効化または削除する

    macOSの画面操作では「ユーザとグループ」から作業用アカウントを削除するのが分かりやすい方法です。検収ログなどを残す必要がある場合は、削除前に管理者へ確認します。

削除前チェック: 自分用アカウントでログインできる、管理者権限がある、必要な検収メモやファイルが残っていない、復旧できる管理者アカウントが少なくとも1つあることを確認します。
削除しない運用もあり: 一時的に残す場合は、作業用アカウントを普段使いしない、Dockやブラウザに個人情報を残さない、パスワードの扱いを管理者側で決める、というルールを明確にします。
First 45 Minutes

初期設定の流れ

ここは全員で画面を合わせて進めます。設定名が見つからない時は、左上のAppleメニューか「システム設定」の検索を使います。

  1. Wi-Fiに接続し、macOSアップデートを確認

    Appleメニュー → システム設定 → 一般 → ソフトウェアアップデート。Windows Updateに相当する場所です。

  2. ローカルユーザー、Touch ID、画面ロックを整える

    Touch IDは便利ですが、パスワードを忘れないようにします。パスワード管理アプリを使う場合は、最初にログインできる状態にしておきます。

  3. FinderとDockを使いやすくする

    Finderのサイドバーに「ホーム」「デスクトップ」「ダウンロード」を表示。Dockはよく使うアプリだけに絞ります。

  4. ターミナルを開き、開発用コマンドを入れる

    Spotlightで「ターミナル」を検索します。以降のコマンドは1行ずつコピーして実行します。

Mac Superpower

Spotlightを使う

Spotlightは、アプリ、ファイル、設定、計算、変換をすばやく呼び出せるMac標準の検索機能です。 Windowsのスタート検索に近いですが、Macでは作業の入口として毎日使います。

Open

⌘ + Spaceで開く

画面中央に検索欄が出ます。アプリ名、ファイル名、設定名を入力し、候補を選んでEnterで開きます。

Apps

アプリ起動に使う

「code」「chrome」「terminal」「activity」など、数文字だけで目的のアプリを開けます。Dockを探す時間が減ります。

Quick

計算と単位変換にも使う

120*1.1、10 USD JPY、32 inch cm のように入力すると、簡単な答えをすぐ確認できます。

探したいもの 入力例 使いどころ
アプリ terminal / vscode / chrome LaunchpadやFinderを開かずに起動する。
設定 software update / firewall / keyboard システム設定の奥にある項目へすばやく移動する。
ファイル README / invoice / .py 保存場所を忘れたファイルを探す。候補を選んでCommandキーを押すと場所を確認できます。
計算・変換 2048/8 / 1 mile km ブラウザや電卓アプリを開かずに軽い確認を済ませる。
最初の練習: ⌘ + Space → terminal → Enter、 次に activity monitor、software update、firewall を検索してみます。
Update & Firewall

更新とファイアウォール

Windows Updateに相当するのはmacOSの「ソフトウェアアップデート」です。 開発ツールを入れる前に、OS更新とネットワーク保護の場所だけ押さえておきます。

Windows Update相当

ソフトウェアアップデート

Appleメニュー → システム設定 → 一般 → ソフトウェアアップデート。セキュリティ更新やmacOS更新をここで確認します。Spotlightで「software update」と検索しても開けます。

Firewall

ファイアウォール

Appleメニュー → システム設定 → ネットワーク → ファイアウォール。外部からMac上のアプリへ接続される通信、つまりインバウンド接続を制御します。

おすすめ: 自宅や外出先のWi-Fiで使うMacは、ファイアウォールをONにしておくと安心です。開発サーバーや共有機能がつながらない時は、ファイアウォールの「オプション」で許可状態を確認します。
アウトバウンド制御について: macOS標準のファイアウォール画面は、アプリが外部へ接続する通信を細かく止める用途ではありません。macOSには低レベルなパケットフィルタ機能もありますが、アプリ単位で分かりやすく制御するものではないため、外向き通信の監視や制御が必要な場合は専用ツールやネットワーク側の制御を検討します。
更新前の注意: 大きなmacOSアップグレードは時間がかかることがあります。バッテリーではなく電源接続、作業ファイル保存、時間に余裕がある時に実行します。

外向き通信を見たい時はLuLu

macOS標準のFirewallでは、アプリごとのアウトバウンド通信を分かりやすく制御できません。 無償で始めやすい選択肢としては、Objective-Seeの LuLu があります。 未知の外向き通信を検知して、アプリ単位で許可/拒否ルールを作れるオープンソースのFirewallです。 最初から必須ではありませんが、AIツールや開発ツールがどこへ通信しているかを確認したくなった時の候補として覚えておくと便利です。

Install

開発ツールを入れる

まずAppleのコマンドラインツール、次にHomebrew、その後に開発ツールを入れます。 コピー前にURLが公式かどうかを必ず確認します。

ターミナルでの約束

パスワード入力中は文字が表示されません。止まって見えても入力できています。知らないコマンドをネットから貼り付ける時は、必ずリーダーに確認します。

Why

Apple コマンドラインツールとは

xcode-select --install で入る、Mac標準の開発用部品です。Git、コンパイラ、ビルドに必要な基本ツールが含まれます。Homebrewや一部の開発ツールが内部で使うため、最初に入れておきます。

Why

Homebrewとは

Mac用のパッケージマネージャーです。brew install で開発ツールをまとめて入れたり、更新したりできます。uv、Go、Node、Git、VS Code、Chromeなどを同じ作法で管理できるようになります。

1. Apple コマンドラインツール最初だけ
xcode-select --install
2. Homebrew公式: brew.sh
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
Apple SiliconのMacの場合: Homebrewの最後に表示される「Next steps」を必ず実行します。多くの場合、/opt/homebrew/bin/brew を使えるようにする設定が表示されます。
3. ブラウザ・エディタ・言語ツールまとめて導入
brew install --cask visual-studio-code google-chrome firefox appcleaner
brew install uv go node git ripgrep
4. Rustrustupで導入
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
. "$HOME/.cargo/env"
rustc --version
cargo --version
Rustを入れる理由: Rust本体を書く予定がなくても、Rust製の開発ツールは cargo install で配布されることがあります。 rustup はRust本体、cargo、rustc をまとめて管理する公式推奨の方法です。 インストール後に cargo が見つからない場合は、ターミナルを開き直します。
5. Pythonはuvで管理例: 最新安定版を導入
uv python install
uv python list
6. Claude Code利用開始時
# Claude Codeを使う準備ができたら実行
curl -fsSL https://claude.ai/install.sh | bash
claude

Claude Codeは公式ドキュメント上ではmacOS 13.0以上、4GB以上のRAM、インターネット接続が要件です。 インストール前に、利用規約、料金、ログ、ソースコードの取り扱い方針を確認してください。

Check

入ったか確認する

ここで全員の画面を揃えます。エラーが出た人は先に進まず、隣の人と同じ表示になるまで確認します。

  • Homebrew: brew --version でバージョンが表示される
  • VS Code: アプリが開ける。必要ならCommand Paletteで Shell Command: Install 'code' command in PATH を実行
  • Python: uv python list でPython一覧が表示される
  • Go: go version が表示される
  • Node: node -v と npm -v が表示される
  • Rust: rustc --version と cargo --version が表示される
  • Git: git --version が表示される
  • Chrome/Firefox: どちらも起動できる
  • AppCleaner: アプリ一覧が表示できる。必要なアプリやセキュリティ関連のものは不用意に消さない

Maintenance

日々の更新作業

開発ツールは入れて終わりではありません。週1回程度、作業前にまとめて更新すると、 セキュリティ修正やCLIツールの不具合修正を取り込みやすくなります。

Homebrew

brewで入れたツール

Homebrew本体の情報を更新し、入れているCLIやアプリを更新します。ChromeやVS Codeなどのcaskアプリも対象です。

Rust

Rust本体

rustup update でRustのツールチェーンを更新します。Rust製ツールをビルドするための土台です。

Cargo

cargoで入れたツール

cargo install で入れたCLIは、基本的に個別の再インストールで更新します。何を入れたかをメモしておくと楽です。

Homebrewの更新週1回の基本
brew update
brew outdated
brew upgrade
brew upgrade --cask
brew cleanup
Rustの更新rustup
rustup update
rustc --version
cargo --version
cargoで入れたCLIの更新例必要なものだけ
cargo install ripgrep --locked
cargo install cargo-update --locked
cargo install-update -a
cargo install-updateについて: cargo install で入れたツールをまとめて更新したい場合は、先に cargo install cargo-update --locked で補助ツールを入れます。 ただし、最初から必須ではありません。Rust製CLIを複数入れるようになってからで十分です。
uvで入れたPythonの確認必要に応じて
uv python list
uv python install
uv本体の更新: この資料ではuvをHomebrewで入れているため、uv本体は brew upgrade で更新されます。 uvで管理しているPythonの追加や確認は、必要なタイミングで uv python list と uv python install を使います。

Trouble Notes

困った時の見方

エラーは珍しいことではありません。どこで止まったか、何と表示されたかを揃えると、解決がかなり速くなります。

command not found: brew

Homebrewの「Next steps」が未実行の可能性があります。ターミナルを開き直す、または表示されたPATH設定を実行します。

インストールが途中で止まった

ネットワーク、プロキシ、管理者権限の可能性があります。エラー画面を消さずに内容を確認します。

アプリが開けない

システム設定 → プライバシーとセキュリティを確認します。出所が分からないアプリは無理に開かず、公式サイトから入手し直します。

何を消してよいか分からない

AppCleanerは便利ですが、セキュリティ製品、同期ツール、よく分からない常駐アプリは削除前に役割を確認します。

参照元

インストールコマンドは、作成時点で公式ドキュメントを確認しています。実施日に再確認するとより安全です。
