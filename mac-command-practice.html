Download

Apacheログを準備する

Elasticが公開しているApache combined log形式のサンプルを使います。 練習用フォルダを作り、その中に apache_logs という名前で保存します。

Folder

練習場所を作る

前回と同じく、ホームディレクトリ直下の ~/practice の中に、ログ分析用フォルダを作ります。

Download

curlで取得する

-L でリダイレクトをたどり、-o で保存ファイル名を指定します。

Check

行数と先頭を見る

ダウンロードできたら wc -l と head で、ファイルが読める状態か確認します。

準備コマンドTerminalで実行
mkdir -p ~/practice/apache-log-analysis
cd ~/practice/apache-log-analysis
curl -L -o apache_logs "https://raw.githubusercontent.com/elastic/examples/master/Common%20Data%20Formats/apache_logs/apache_logs"
wc -l apache_logs
head -n 3 apache_logs
ポイント: このページの演習は、ファイル名が apache_logs である前提です。 別名で保存した場合は、コマンド中のファイル名も置き換えてください。
Log Format

ログの列を読む

Apache combined logは、人間には少し長く見えます。 まずは空白区切りでよく使う列だけを覚えます。

1行の例Apache combined log
83.149.9.216 - - [17/May/2015:10:05:03 +0000] "GET /presentations/logstash-monitorama-2013/images/kibana-search.png HTTP/1.1" 200 203023 "http://semicomplete.com/presentations/logstash-monitorama-2013/" "Mozilla/5.0 ..."
空白区切りの位置意味抜き出し例
1列目アクセス元IPcut -d ' ' -f 1 apache_logs
4列目日時の前半cut -d ' ' -f 4 apache_logs
6列目HTTPメソッドcut -d ' ' -f 6 apache_logs
7列目リクエストされたパスcut -d ' ' -f 7 apache_logs
9列目ステータスコードcut -d ' ' -f 9 apache_logs
10列目レスポンスサイズcut -d ' ' -f 10 apache_logs
注意: User-Agentには空白が含まれるため、cut -d ' ' だけで正確に列として扱うのは難しいです。 この教材では、まずIP、パス、ステータスコードの分析に集中します。
Final Report

最後に調査メモを作る

コマンドを覚えるだけでなく、調べた結果をファイルに残すところまでを練習します。 実務では、再現できるコマンドと保存された結果があると共有しやすくなります。

Top IP

アクセス元の上位

cut、sort、uniq -c、sort -nr、head をつなぎます。

Errors

404と500番台

grep で絞り込み、パスを抜き出して、どこでエラーが多いかを見ます。

Save

結果を保存する

> で上書き保存、>> で追記保存。調査結果をメモとして残します。

仕上げの例調査結果を保存
cut -d ' ' -f 1 apache_logs | sort | uniq -c | sort -nr | head > top-ip.txt
grep " 404 " apache_logs | cut -d ' ' -f 7 | sort | uniq -c | sort -nr | head > top-404-path.txt
grep " 50[0-9] " apache_logs > server-errors.log
cat top-ip.txt top-404-path.txt > investigation-summary.txt
