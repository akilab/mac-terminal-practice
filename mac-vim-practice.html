Why Vim

突然開いても、慌てないために。

vi/vimは古く見えても、UNIX系環境では今でもよく出てきます。普段のエディタにしなくても、最低限の読み書きと終了方法を知っているだけで安心です。

どこでも

サーバーに入っていることが多い

GUIエディタがない環境でも、vi や vim だけは使える場面があります。

突然

Git操作で開くことがある

git commit でメッセージ編集画面が開いた時、保存して閉じられれば先に進めます。

最低限

全部覚えなくてよい

最初は Esc、保存、終了、少しの移動と編集だけで十分です。

Modes

Vimは、いまのモードで動きが変わる。

混乱の正体は、多くの場合「文字を打つモード」と「命令を出すモード」が分かれていることです。迷ったらまず Esc でNormal modeへ戻ります。

Normal 移動、削除、コピー、貼り付けなどの命令を押す場所。
Insert 文字を入力する場所。左下に -- INSERT -- と出ます。
Command-line : から保存、終了、置換、設定などを入力する場所。画面下に入力され、Enter で実行、Esc でキャンセルします。
Visual 範囲選択をする場所。この教材では名前だけ知っていれば十分です。
合言葉は「まずEsc」

文字が入らない、変な動きをした、コマンドを打てない。そんな時は Esc を1回か2回押してNormal modeに戻ります。左下の -- INSERT -- が消えたら、命令を受け付ける状態です。

Setup

安全な練習ファイルを作る。

練習場所は ~/practice/vim-lab です。ホームディレクトリ配下の小さなテキストだけを触るので、何度失敗しても大丈夫です。

Terminalで実行: 練習場所とサンプルを作る
mkdir -p ~/practice/vim-lab
cd ~/practice/vim-lab
printf 'Vim practice memo\napple banana apple\nsave before quit\n' > memo.txt
cat memo.txt

この準備コマンドを再実行すると、memo.txt は作り直されます。練習を最初からやり直したい時だけ実行します。

Terminalで実行: 練習ファイルをVimで開く
cd ~/practice/vim-lab
vim memo.txt
ここから先はVim内で押すキーも出てきます

vim memo.txt でVimを開いた後、:wq や dd はTerminalではなくVimの画面内で押します。各ブロックの見出しに「Terminalで実行」か「Vim内で押すキー」と書いてあります。

「Vim内で押すキー」はコピーして貼り付けるものではありません。1行ずつ、実際にキーボードで押してください。

Save and Quit

終了できれば、怖さは半分になる。

まず Esc でNormal modeに戻り、: から命令を入力します。初心者は保存して終了する :wq を中心に覚えれば大丈夫です。:q! は保存していない変更を捨てるため、困った時の非常口として扱います。

Vim内で押すキー: 保存して終了
Esc
:wq
Enter
キー意味注意
EscNormal modeへ戻る迷った時の最初の一手。左下の -- INSERT -- が消える。
:q終了する変更があると終了できない。安全に止まる。
:q!保存せず終了する変更を捨てる。必要な編集まで消えるので意味を分かって使う。
:w保存するVimは開いたまま。続けて編集できる。
:wq保存して終了する最初に一番使う形。
ZZ保存して終了するNormal modeで押す短縮形。紹介程度でOK。

Enter

入力を始めるキーはいくつかあります。左下に -- INSERT -- が出たら文字入力中です。入力が終わったら Esc で戻ります。

Yank and Put

Vimのコピーは、yankとput。

Vimではコピーを yank、貼り付けを put と呼びます。まずは行単位の yy と p を覚えます。

Vim内で押すキー: 1行コピーして下に貼る

Substitute

置換は強い。最初は確認付きで。

:s は行内置換、:%s はファイル全体の置換です。% は「ファイル全体」、最後の g は「その行の中に同じ対象が複数あれば全部」を意味します。全置換は強力なので、慣れるまでは gc を付けて1件ずつ確認します。

Vim内で押すキー: 1行だけ置換する
Esc
:s/apple/orange/g
Enter
Vim内で押すキー: ファイル全体を確認付きで置換する
Esc
:%s/apple/orange/gc
Enter
形意味安全メモ
:s/old/new/g今いる1行だけ置換g があるので、その1行内の対象をすべて置換する。
:%s/old/new/gファイル全体を一括置換% があるので、全行が対象になる。実行前に対象をよく見る。
:%s/old/new/gcファイル全体を確認しながら置換c はconfirm。y で置換、n でスキップ。
置換結果を確認するまで、追加保存しない

置換後、まだ保存していなければ u で戻せます。結果を確認するまでは追加で :w せず、慣れるまでは :%s/old/new/gc で1件ずつ確認します。

Troubleshooting

困った時は、状態を戻してから考える。

Vimで詰まる時は、壊れたのではなく「今いるモード」や「保存状態」が分からなくなっていることが多いです。まずは深呼吸して Esc です。

状況まずやること次に見ること
何を押しても思った通りにならないEsc を1回か2回押す左下の -- INSERT -- が消えたか見る。
保存して終わりたいEsc、:wq、Enter最初はこれを体に入れる。
変更を捨てて終わりたいEsc、:q!、Enter:q! は変更を捨てる。必要な内容がない時だけ。
E37: No write since last change が出た保存するなら :wq捨てるなら :q!。Vimが安全に止めてくれている。
swapファイルらしい表示が出た慌てて連打しない同じファイルを別画面で開いていないか、前回クラッシュしていないか読む。
Git commitでVimが開いたi でメッセージを書くEsc、:wq、Enter で保存終了してcommitを進める。
Git commitを中止したいメッセージを書かずに Esc、:q!、Enter保存せず閉じるため、commitは作られません。
Vim内で押すキー: Git commit画面の最低限
i
Add short commit message
Esc
:wq
Enter
Vim内で押すキー: Git commitを中止する
Esc
:q!
Enter
swapファイル表示は、読んでから選ぶ

Vimは編集中の復旧情報をswapファイルとして持つことがあります。別のVimで同じファイルを開いている、前回異常終了した、などの可能性があります。分からない時は画面の説明を読み、必要なら詳しい人に状況を見せます。
