Mac Process / Job Practice

動いているものを、見つけて止める。

プロセス、ジョブ、ポート、ネットワーク状態、ログ監視を学びます。 開発サーバーが終わらない、ポートが使用中、どのコマンドを止めればよいか分からない、という場面に備える教材です。

プロセスから学ぶ ポート確認へ
MacBook-Air % ps aux | grep node
akr 4821 0.1 node server.js

MacBook-Air % lsof -nP -iTCP:3000 -sTCP:LISTEN
node 4821 akr 22u IPv6 TCP *:3000 (LISTEN)

MacBook-Air % kill 4821
MacBook-Air % tail -f app.log
プロセス 止め方 ジョブ 確認 ポート ネットワーク ログ 困った時
Process

プロセスとは、実行中のプログラム

コマンドを実行すると、多くの場合プロセスが作られます。プロセスには番号があり、それをPIDと呼びます。

Process

動いているプログラム

node、python3、zsh、ブラウザなど、実行中のものはプロセスです。

PID

プロセス番号

OSがプロセスに付ける番号です。終了させる時は、このPIDを指定することがあります。

Parent

親子関係がある

Terminalのシェルからコマンドを起動すると、シェルが親、そのコマンドが子プロセスになります。

今のシェルとPIDを見る
echo $$
ps -p $$
ps -f -p $$
まずPIDを怖がらない

PIDは「動いているものに付いた番号」です。何かを止めたい時は、まず対象プロセスを見つけ、PIDを確認し、必要なら終了シグナルを送ります。

Stop

まずは安全な止め方から

終わらないコマンドを止める時、最初に使うのは Ctrl+C です。強制終了は最後の手段にします。

操作意味使う場面
Ctrl+C今動いている前面のコマンドに終了を依頼する開発サーバー、tail、pingなどを止める
Ctrl+Z今動いているコマンドを一時停止する止めるのではなく一時停止したい時
kill PID指定したプロセスに終了を依頼する別Terminalから止めたい時
kill -9 PID強制終了する通常のkillで止まらない時の最後の手段
Ctrl+Cを試しやすいコマンド
ping 127.0.0.1
止め方の優先順位

まず Ctrl+C、次に kill PID、それでも止まらない時だけ kill -9 PID を検討します。kill -9 は片付け処理をさせずに止めるため、常用しません。

Jobs

前面と背面を切り替える

シェルにはジョブという考え方があります。今のTerminalで動かしているコマンドを、一時停止したり背面で動かしたりできます。

Foreground

前面

Terminalを占有している状態です。通常コマンドは前面で動きます。

Background

背面

Terminal入力を返しながら、裏で動いている状態です。末尾に & を付けて起動できます。

Job

シェル内の管理単位

jobs で一覧を見て、fg や bg で前面・背面を切り替えます。

ジョブ操作を確認する
sleep 300
# Ctrl+Z で一時停止
jobs
fg
# Ctrl+C で終了
最初から背面で動かす
sleep 300 &
jobs
fg %1
Ctrl+C と Ctrl+Z は違う

Ctrl+C は終了を依頼します。Ctrl+Z は一時停止です。一時停止したジョブは残るので、fg で戻すか、必要なら終了します。

Inspect

動いているプロセスを確認する

何が動いているか分からない時は、ps、pgrep、top で確認します。

プロセス一覧から探す
ps aux | head
ps aux | grep node
ps aux | grep python
名前でPIDを探す
pgrep -fl node
pgrep -fl python
pgrep -fl zsh
負荷が高いプロセスを見る
top -o cpu
top -o mem
ps aux | grep の注意

grep 自身も検索結果に出ることがあります。より見やすくしたい時は pgrep -fl 名前 を使うと、PIDとコマンド名を探しやすくなります。

Port

どのポートを誰が使っているか調べる

開発中によく出る「Port 3000 is already in use」は、そのポートを何かが使っているという意味です。多くの場合は前に起動した開発サーバーですが、COMMAND名を見てから判断します。

ポート3000を使っているプロセスを探す
lsof -nP -iTCP:3000 -sTCP:LISTEN
目的

lsof は「誰が何を開いているか」を見る

lsof は list open files の略です。UNIX/macOSでは通常のファイル、ディレクトリ、実行ファイル、端末、ネットワーク接続などを「開いているもの」として扱えます。ネットワーク専用ではなく、オプションでネットワークに絞って使います。

まず見る

開発サーバーの衝突を確認する

Port 3000 is already in use と出たら、まず lsof -nP -iTCP:3000 -sTCP:LISTEN で待ち受け中のプロセスを確認します。基本は 3000 の部分だけを、エラーに出た番号へ置き換えます。

何も指定しない lsof は情報が多すぎる

lsof -nP | head は、開いているもの全体の先頭だけを表示します。先頭には /proc/1/cwd や /proc/1/exe のような、プロセスが参照しているファイルやディレクトリが出ることがあります。ここにポートが見えなくても正常です。

lsofを段階的に絞り込む
# 開いているもの全体を見る。量が多いので先頭だけ確認する
lsof -nP | head

# ネットワーク関係だけに絞る
lsof -nP -i

# TCPで待ち受け中のポートだけに絞る
lsof -nP -iTCP -sTCP:LISTEN

# 3000番ポートだけに絞る
lsof -nP -iTCP:3000 -sTCP:LISTEN
結果が空でも、コマンド失敗とは限らない

lsof -nP -i や lsof -nP -iTCP -sTCP:LISTEN で何も出ない場合、その環境で表示できるネットワーク接続や待ち受けプロセスが無いだけかもしれません。WSLでは見え方がMacと異なることもあります。開発サーバーを起動してから、対象ポートを指定して確認すると理解しやすくなります。

部分意味初心者向けの読み方
-nP名前解決を省略する-n はIPをホスト名に変換しない、-P はポート番号をサービス名に変換しない指定です。表示が速く、3000 のような番号のまま読めます。
-iネットワーク関連に絞るlsof は本来いろいろな「開いているもの」を表示するため、ポートを見たい時は -i でネットワークに絞ります。
-iTCP:3000TCPの3000番ポートに絞るTCP:3000 で「TCP通信の3000番ポートだけ」を探します。番号だけ変えれば、他のポートを確認できます。
-sTCP:LISTEN待ち受け中だけに絞るLISTEN は接続を待っている状態です。開発サーバーがポートを占有しているか知りたい時は、ここを見るのが近道です。
出力例と見る場所
COMMAND   PID USER   FD   TYPE             DEVICE SIZE/OFF NODE NAME
node     4821 akr   22u  IPv6 0x123456789abcdef0      0t0  TCP *:3000 (LISTEN)
列見るポイント
COMMAND動いているコマンド名です。node、python、ruby などが出ます。
PIDプロセス番号です。止める時はこの番号を kill PID に渡します。
USERそのプロセスを起動したユーザーです。自分のユーザー名なら、自分が起動した開発サーバーの可能性が高いです。
NAME*:3000 (LISTEN) のように出ていれば、3000番ポートで接続待ちをしています。
よく使う開発ポートを確認する
lsof -nP -iTCP:3000 -sTCP:LISTEN
lsof -nP -iTCP:5173 -sTCP:LISTEN
lsof -nP -iTCP:8000 -sTCP:LISTEN
lsof -nP -iTCP:8080 -sTCP:LISTEN
慣れないうちは1行ずつ実行する

エラーに出ているポート番号が分かっている場合は、その番号の行だけで十分です。例えば 5173 が使用中なら、-iTCP:5173 の行だけを実行します。

まず覚える実用パターン
# 3000番を使っている待ち受けプロセスを探す
lsof -nP -iTCP:3000 -sTCP:LISTEN

# 番号だけ変えて別ポートを見る
lsof -nP -iTCP:5173 -sTCP:LISTEN

# 表示されたCOMMANDとPIDを確認し、止めてよい開発サーバーなら実行する例
kill 4821

# 止まったかもう一度確認する
lsof -nP -iTCP:3000 -sTCP:LISTEN
PIDを指定して終了する例
kill 4821
# 止まらない時だけ
kill -9 4821
lsof の読み方

4821 は例のPIDです。まず COMMAND、PID、NAME を見ます。止める前に、アプリ名やユーザー名から本当に自分が止めてよいプロセスか確認しましょう。

Network

ネットワーク状態を確認する

通信できない時は、名前解決、疎通、HTTP応答、待ち受け状態を分けて確認します。

外向き

相手へ届くか

ping、curl -I、nc で、相手サーバーへの疎通やHTTP応答を確認します。

内向き

自分が待ち受けているか

lsof で、自分のMac上でどのプロセスがどのポートを使っているか確認します。nc は接続できるかを試す道具なので、役割が違います。

基本的な疎通確認
ping -c 4 8.8.8.8
ping -c 4 github.com
curl -I https://github.com
ポートへ接続できるか確認する
nc -vz github.com 443
nc -vz localhost 3000
自分のIPや経路を見る
ifconfig
netstat -rn
scutil --dns | head -40
確認を分ける

ping が通らなくてもHTTPが通ることはあります。会社ネットワークではICMPが止められている場合もあるため、curl -I や nc -vz も使って確認します。

Logs

ログを追いかける

サーバーを起動した後は、ログを見ながら動作確認する場面が増えます。tail -f は基本の道具です。

ログの末尾を見る
tail app.log
tail -n 50 app.log
tail -f app.log
ログを追いながら絞り込む
tail -f app.log | grep ERROR
tail -f app.log | grep -E "ERROR|WARN"
tail -f の終わり方

tail -f はログを追い続けるコマンドなので、自然には終わりません。終了する時は Ctrl+C を押します。

Troubleshooting

よくある困りごと

開発中に多い状況を、確認する順番で整理します。焦った時ほど、状態確認から始めます。

状況まず見るもの代表コマンド
コマンドが終わらない前面で動いているかCtrl+C
一時停止してしまったジョブ一覧jobs, fg
ポートが使用中そのポートのLISTENプロセスlsof -nP -iTCP:3000 -sTCP:LISTEN
CPUが高い負荷の高いプロセスtop -o cpu
通信できないHTTP応答、ポート接続curl -I URL, nc -vz host port
ログが流れ続ける追跡中のコマンドCtrl+C
開発サーバーが残っている時の確認手順
lsof -nP -iTCP:3000 -sTCP:LISTEN
pgrep -fl node
# 表示されたPIDを確認してから実行する例
kill 4821
lsof -nP -iTCP:3000 -sTCP:LISTEN
教材での安全方針

このページでは、削除系の危険な操作は扱いません。プロセス終了も、まず対象を確認し、通常の kill を使い、kill -9 は最後の手段として扱います。
